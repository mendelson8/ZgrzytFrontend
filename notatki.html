0. Podstawy WPF: Właściwości Okna (Window)

Jak zmienić nazwę, wymiary i logo aplikacji?

Wszystkie podstawowe ustawienia Twojej aplikacji definiujesz w głównym tagu <Window> na samej górze pliku XAML. To tutaj ustalasz, co wyświetla się na pasku tytułowym, jak duże jest okno na starcie i czy użytkownik ma prawo je minimalizować lub rozciągać.

  • Title - Nazwa okienka widoczna na samej górze.
  • Icon - Ścieżka do pliku z logo (np. .png lub .ico). Ważne: plik graficzny musi być dodany do projektu w Visual Studio, a jego Build Action najlepiej ustawić na Resource.
  • Width / Height - Wymiary startowe, z jakimi okno się uruchomi.
  • MinWidth / MinHeight - Blokada przed zbytnim pomniejszeniem okna. Przydatne, by interfejs się nie "zgniótł".
  • WindowStartupLocation - CenterScreen sprawia, że okno zawsze pojawia się na środku monitora, a nie w losowym rogu.
  • ResizeMode - Ustawienie na NoResize zabrania użytkownikowi powiększania/pomniejszania okna (ukrywa przycisk maksymalizacji).
Plik: MainWindow.xaml (Główny tag okna)
<Window x:Class="MojaAplikacja.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"

        <!-- 1. NAZWA I LOGO -->
        Title="Moja Super Aplikacja"
        Icon="logo.png"

        <!-- 2. WYMIARY STARTOWE -->
        Width="800" Height="600"

        <!-- 3. WYMIARY MINIMALNE (Zabezpieczenie przed zgnieceniem UI) -->
        MinWidth="500" MinHeight="400"

        <!-- 4. ZACHOWANIE OKNA -->
        WindowStartupLocation="CenterScreen"
        ResizeMode="CanResize">

    <!-- Tutaj wchodzi reszta interfejsu np. <Grid> -->

</Window>

0.5. Podstawy WPF: Kolory (Background i Foreground)

Jak poprawnie kolorować interfejs?

W WPF nie używamy właściwości Color. Zamiast tego mamy dwie główne właściwości:

  • Background - Zmienia kolor tła kontrolki (np. tło przycisku, siatki, okna).
  • Foreground - Zmienia kolor tego, co jest "na wierzchu", czyli najczęściej kolor tekstu.

Wartości możemy podawać na trzy sposoby:

  1. Nazwa koloru: Wbudowane nazwy z języka angielskiego (np. Red, LightBlue, Transparent).
  2. Kod HEX (ARGB): Podobnie jak w CSS, np. #FF0000 to czerwony. Ważne: W WPF często podaje się 8 znaków zamiast 6 (np. #80FF0000). Pierwsze dwie litery (80) to kanał Alpha, czyli przezroczystość (gdzie FF to w pełni widoczny, a 00 to całkowicie niewidzialny, 80 to około 50%).
  3. Pędzle (Brushes): Jeśli chcesz zrobić coś bardziej zaawansowanego, np. płynne przejście kolorów (gradient), używasz np. LinearGradientBrush.
Plik: MainWindow.xaml (Przykłady zmiany kolorów)
<!-- 1. TŁO OKNA: Zwykły kod HEX (Szare tło) -->
<Window Background="#F0F0F0">

    <StackPanel Margin="20">

        <!-- 2. KOLORY NAZWANE: Zwykły tekst -->
        <TextBlock Text="Uwaga, błąd systemu!"
                   Foreground="Red"
                   Background="Black"
                   FontSize="20"
                   Padding="10"/>

        <!-- 3. PRZEZROCZYSTOŚĆ W HEX (Kanał Alpha) -->
        <Button Content="Półprzezroczysty zielony przycisk"
                Background="#8000FF00"
                Foreground="#FFFFFF"
                Margin="0,20,0,0"
                Padding="10"/>

        <!-- 4. ZAADWANSOWANE: Gradient (LinearGradientBrush) -->
        <Button Content="Przycisk z Gradientem" Foreground="White" Margin="0,20,0,0" Padding="10">
            <Button.Background>
                <LinearGradientBrush StartPoint="0,0" EndPoint="1,1">
                    <GradientStop Color="Blue" Offset="0.0" />
                    <GradientStop Color="Purple" Offset="1.0" />
                </LinearGradientBrush>
            </Button.Background>
        </Button>

    </StackPanel>
</Window>

WPF: Contact Manager - Głęboka Analiza

XAML potrafi być przytłaczający. Rozłóżmy go na czynniki pierwsze, żebyś rozumiał każdą linijkę, którą wpisujesz.

1. Ustawienia Okna i Układ (Grid) w XAML

Po co nam Grid i RowDefinitions?

Wymaganie z PDF mówi jasno: "Pasek menu na górze okna, reszta okna podzielona na zakładki". W WPF nie używamy pozycjonowania pikselowego. Używamy <Grid>, który działa jak niewidzialna tabela.

Dzielimy okno na dwa wiersze:

  • Height="Auto" - Pierwszy wiersz dopasuje swoją wysokość do zawartości (czyli będzie miał dokładnie taką wysokość jak Pasek Menu).
  • Height="*" - Znak gwiazdki to "weź całą resztę wolnego miejsca". Tu wrzucimy zakładki (TabControl).
Plik: MainWindow.xaml (Fragment Głównego Układu)
<Window x:Class="ContactManager.MainWindow"
        Title="Contact Manager"
        Height="600" Width="800"
        MinWidth="500" MinHeight="500"
        WindowStartupLocation="CenterScreen">

    <Grid>
        <!-- Definicja naszej tabeli (2 wiersze) -->
        <Grid.RowDefinitions>
            <RowDefinition Height="Auto"/>
            <RowDefinition Height="*"/>
        </Grid.RowDefinitions>

        <!-- Wiersz 0: Menu -->
        <Menu Grid.Row="0">
            <MenuItem Header="File">
                <MenuItem Header="Exit" Click="MenuItem_Exit" />
            </MenuItem>
        </Menu>

        <!-- Wiersz 1: Zakładki zajmujące resztę ekranu -->
        <TabControl Grid.Row="1">
            <TabItem Header="List"> <ListBox/> </TabItem>
            <TabItem Header="Grid"> <DataGrid/> </TabItem>
        </TabControl>
    </Grid>
</Window>

2. Powiadomienia Widoku (INotifyPropertyChanged)

Dlaczego ObservableCollection nie wystarcza?

Samo użycie ObservableCollection<Contact> informuje widok tylko o tym, że element został dodany lub usunięty z listy. Nie powiadamia o edycji wartości (np. zmianie nazwiska). Używamy do tego interfejsu INotifyPropertyChanged.

Plik: Contact.cs
using System.ComponentModel;
using System.Runtime.CompilerServices;

public class Contact : INotifyPropertyChanged
{
    private string name;

    public string Name
    {
        get { return name; }
        set
        {
            name = value;
            OnPropertyChanged();
        }
    }

    public event PropertyChangedEventHandler PropertyChanged;
    protected void OnPropertyChanged([CallerMemberName] string propertyName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propertyName));
    }
}

3. Magia: ComboBox ze zmiennymi typu Enum

Jak wyciągnąć "Male" i "Female" do listy rozwijanej?

Aby uniknąć brzydkiego kodu w C#, korzystamy z ObjectDataProvider, ładując nasz Enum bezpośrednio do Window.Resources i bindując jako ItemsSource.

Plik: AddContactWindow.xaml
<Window.Resources>
    <!-- Pobiera wartości (GetValues) z typu Enum, a dokładnie z naszego local:Gender -->
    <ObjectDataProvider x:Key="GenderList" MethodName="GetValues" ObjectType="{x:Type TypeName=System:Enum}">
        <ObjectDataProvider.MethodParameters>
            <x:Type TypeName="local:Gender"/>
        </ObjectDataProvider.MethodParameters>
    </ObjectDataProvider>
</Window.Resources>

<Grid>
    <!-- ComboBox z podpiętą listą wartości Enum oraz powiązaniem wyboru z właściwością obiektu -->
    <ComboBox ItemsSource="{Binding Source={StaticResource GenderList}}"
              SelectedItem="{Binding Path=Gender}" />
</Grid>

4. Kolorowanie co drugiego wiersza (AlternationIndex)

Dlaczego używamy Style Triggers?

By wiersze automatycznie kolorowały się na zmianę (niezależnie od tego, jak sortujemy czy usuwamy dane ze środka), ustawiamy w ListBox atrybut AlternationCount="2". Resztę załatwia Trigger.

Plik: MainWindow.xaml
<!-- Kluczowe: AlternationCount włącza liczenie 0, 1, 0, 1... -->
<ListBox ItemsSource="{Binding Contacts}" AlternationCount="2">

    <ListBox.ItemContainerStyle>
        <Style TargetType="ListBoxItem">
            <Style.Triggers>
                <!-- Wyzwalacz dla wierszy parzystych (0) -->
                <Trigger Property="ItemsControl.AlternationIndex" Value="0">
                    <Setter Property="Background" Value="#FFAFC5FF" />
                </Trigger>

                <!-- Wyzwalacz dla wierszy nieparzystych (1) -->
                <Trigger Property="ItemsControl.AlternationIndex" Value="1">
                    <Setter Property="Background" Value="#FF75A1FF" />
                </Trigger>
            </Style.Triggers>
        </Style>
    </ListBox.ItemContainerStyle>

</ListBox>

5. Zaawansowana Walidacja (IValidation)

Architektura rozszerzalna przez pliki DLL

Wykorzystujemy interfejs IValidation oraz dziedziczymy po ValidationRule, aby aplikacja pozwalała na wstrzykiwanie nowych zasad walidacji bez modyfikowania głównego kodu okna.

Plik: PhoneValidationRule.cs
public interface IValidation
{
    string Name { get; }
    string Description { get; }
}

public class PhoneValidationRule : ValidationRule, IValidation
{
    public string Name => "Format Telefonu";
    public string Description => "Format musi wynosić XXX-XXX-XXX.";

    public override ValidationResult Validate(object value, CultureInfo cultureInfo)
    {
        string phone = value as string;
        if (!System.Text.RegularExpressions.Regex.IsMatch(phone, @"^\d{3}-\d{3}-\d{3}$"))
            return new ValidationResult(false, Description);

        return ValidationResult.ValidResult;
    }
}

7. Konwertery (Ocena na gwiazdki i kolor RGB)

Jak przetłumaczyć liczbę 7.5 na ocenę w gwiazdkach i obliczyć kolor RGB?

W WPF nie możemy wpisać matematyki do XAML-a. Pisząc kod, używamy interfejsu IValueConverter, który pośredniczy pomiędzy daną wejściową (np. ocena w Double) a daną wyjściową dla UI (np. ciąg gwiazdek "★★★☆☆" lub kolor pędzla SolidColorBrush).

  • RatingToStarsConverter: Tworzy napis ze znakiem "★" oraz z dopełnieniem "☆" na podstawie formuły Math.Clamp.
  • AverageRatingToStarsConverter: Oblicza średnią ze zbioru ICollection<Chapter> dla konkretnej książki.
  • RatingToColorConverter: Oblicza dynamicznie kanały Red i Green zgodnie z zadaniem (wartości od 0 do 192).
Plik: Converters.cs
using System;
using System.Collections.Generic;
using System.Globalization;
using System.Linq;
using System.Windows.Data;
using System.Windows.Media;
using Potter.API;

namespace MiNIPotter
{
    public class RatingToStarsConverter : IValueConverter
    {
        public object Convert(object value, Type targetType, object parameter, CultureInfo culture)
        {
            if (value is double rating)
            {
                int fullStars = (int)Math.Clamp(rating, 0, 10);
                return new string('★', fullStars) + new string('☆', 10 - fullStars);
            }
            return "";
        }

        public object ConvertBack(object value, Type targetType, object parameter, CultureInfo culture) => throw new NotImplementedException();
    }

    // Analogicznie dla kolejnych konwerterów z polecenia...
}

8. Główny Layout XAML (Wszystko w jednym)

Kluczowe rozwiązania w kodzie:

  • Import Słownika: <ResourceDictionary Source="Styles/Colors/Colors.xaml"/> wstrzykuje kolory, pozwalając na użycie {DynamicResource PrimaryBackgroundBrush}.
  • Zewnętrzna Czcionka: Zdefiniowana jako <FontFamily x:Key="HarryPotterFont">./#Harry Potter</FontFamily>. Odnosi się to bezpośrednio do pliku TTF w głównym katalogu projektu.
  • Formatowanie Daty: StringFormat='{}{0:d MMMM yyyy}' wymusza specyficzny, pełny format daty wymagany w punkcie dla kafelków z okładkami.
  • Expander (Rozdziały): Zamknęliśmy element wewnątrz `Expander`, gdzie nagłówek "Header" przejmuje tytuł rozdziału, a jego rozwinięcie ujawnia szczegółowe opisy i dynamicznie pokolorowane gwiazdki.

Dodatek: Jak zrobić To-Do List w WPF

Zbudowanie prostej listy zadań to świetny sposób na utrwalenie wiedzy. Wykorzystamy tu dynamiczną kolekcję, bindowanie właściwości IsCompleted pod CheckBoxa oraz magię DataTriggera do przekreślania wykonanych zadań.

9. Projekt: Prosta Lista Zadań (To-Do List)

Krok 1: Model Danych (Zadanie)

Wyobraź sobie, że klasa ToDoItem to pojedyncza karteczka z zadaniem. Ma swój tekst (Title) i informację, czy jest odhaczona (IsCompleted).

Dlaczego używamy INotifyPropertyChanged? Przeglądarka (czy okno WPF) jest "ślepa" na zmiany zmiennych w tle. Kiedy zmienisz IsCompleted na true, okno o tym nie wie. Ten interfejs działa jak megafon. Kiedy włączamy checkboxa, uruchamia się OnPropertyChanged(), co krzyczy do interfejsu: "Hej! Mój status się zmienił, odrysuj mnie na nowo!".

Plik: ToDoItem.cs
using System.ComponentModel;
using System.Runtime.CompilerServices;

public class ToDoItem : INotifyPropertyChanged
{
    private string title;
    private bool isCompleted;

    public string Title
    {
        get => title;
        set { title = value; OnPropertyChanged(); }
    }

    public bool IsCompleted
    {
        get => isCompleted;
        set
        {
            isCompleted = value;
            // Kiedy zaznaczasz checkboxa, ta linijka alarmuje XAML o zmianie!
            OnPropertyChanged();
        }
    }

    // Wymagany kod interfejsu (kopiuj-wklej)
    public event PropertyChangedEventHandler PropertyChanged;
    protected void OnPropertyChanged([CallerMemberName] string propName = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(propName));
    }
}

Krok 2: Widok (XAML) - Zrozumieć "Ścianę Kodu"

Najtrudniejszy element to <DataTemplate>. Działa on jak foremka do ciastek. Ponieważ ListBox to lista, która może mieć 5, 50 albo 1000 elementów, nie piszemy kodu dla każdego z nich osobno. Tworzymy jedną "foremkę" (DataTemplate), a WPF wycina z niej wygląd dla każdego zadania na liście.

Druga magia to <DataTrigger>. Działa to jak system IF-THEN: "JEŚLI zadanie ma IsCompleted=True, WTEDY weź element nazwany 'TaskText' i przekreśl go".

Plik: MainWindow.xaml
<Window x:Class="ToDoApp.MainWindow"
        Title="My To-Do List" Height="450" Width="400">
    <Grid Margin="10">

        <!-- 1. PODZIAŁ OKNA NA GÓRĘ I DÓŁ -->
        <Grid.RowDefinitions>
            <RowDefinition Height="Auto"/> <!-- Auto: zajmuje tylko tyle miejsca ile trzeba na wpisanie zadania -->
            <RowDefinition Height="*"/>    <!-- Gwiazdka (*): bierze CAŁĄ resztę okna na listę -->
        </Grid.RowDefinitions>

        <!-- 2. PASEK DODAWANIA (Zajmuje Grid.Row="0") -->
        <Grid Grid.Row="0" Margin="0,0,0,10">
            <Grid.ColumnDefinitions>
                <ColumnDefinition Width="*"/>    <!-- TextBox rozpycha się na maksa -->
                <ColumnDefinition Width="Auto"/> <!-- Przycisk 'Dodaj' bierze tylko szerokość swojego tekstu -->
            </Grid.ColumnDefinitions>

            <TextBox x:Name="NewTaskTextBox" Grid.Column="0" Margin="0,0,10,0" Padding="5" FontSize="16"/>
            <Button Grid.Column="1" Content="Dodaj" Click="AddBtn_Click" Width="80"/>
        </Grid>

        <!-- 3. LISTA ZADAŃ (Zajmuje Grid.Row="1") -->
        <ListBox Grid.Row="1" ItemsSource="{Binding Tasks}" HorizontalContentAlignment="Stretch">
            <ListBox.ItemTemplate>
                <!-- TO JEST NASZA "FOREMKA" NA POJEDYNCZE ZADANIE -->
                <DataTemplate>
                    <Grid Margin="0,5">
                        <Grid.ColumnDefinitions>
                            <ColumnDefinition Width="Auto"/> <!-- Miejsce na mały kwadracik CheckBoxa -->
                            <ColumnDefinition Width="*"/>    <!-- Długi tekst zadania -->
                            <ColumnDefinition Width="Auto"/> <!-- Czerwony krzyżyk do usuwania -->
                        </Grid.ColumnDefinitions>

                        <!-- CheckBox powiązany z IsCompleted. Jak go klikniesz, poleci info do klasy C# -->
                        <CheckBox Grid.Column="0"
                                  IsChecked="{Binding IsCompleted, UpdateSourceTrigger=PropertyChanged}"
                                  VerticalAlignment="Center"/>

                        <!-- Tekst zadania. Zauważ, że daliśmy mu imię (x:Name="TaskText").
                             Bez tego imienia WPF nie wiedziałby, co ma przekreślić! -->
                        <TextBlock x:Name="TaskText" Grid.Column="1"
                                   Text="{Binding Title}" Margin="10,0,0,0" FontSize="16"/>

                        <!-- Przycisk usuwania -->
                        <Button Grid.Column="2" Content="X" Foreground="Red" Click="DeleteBtn_Click"/>
                    </Grid>

                    <!-- WYZWALACZE (TRIGGERS): Reagują na to, co robisz -->
                    <DataTemplate.Triggers>
                        <!-- TŁUMACZENIE: Jeśli zmienna 'IsCompleted' w C# ulegnie zmianie na 'True'... -->
                        <DataTrigger Binding="{Binding IsCompleted}" Value="True">

                            <!-- ...TO zrób dwie rzeczy obiektowi o nazwie "TaskText" (naszemu tekstowi): -->
                            <!-- 1. Przekreśl go (Strikethrough) -->
                            <Setter TargetName="TaskText" Property="TextDecorations" Value="Strikethrough"/>

                            <!-- 2. Zmień kolor na szary -->
                            <Setter TargetName="TaskText" Property="Foreground" Value="Gray"/>
                        </DataTrigger>
                    </DataTemplate.Triggers>

                </DataTemplate>
            </ListBox.ItemTemplate>
        </ListBox>
    </Grid>
</Window>

Krok 3: Logika (C#) - Skąd XAML bierze dane?

Sam XAML to tylko "pusta muszla". W kodzie C# używamy DataContext = this;, żeby powiedzieć XAML-owi: "Szukaj zmiennych, takich jak `Tasks`, w tym pliku". Oraz oczywiście obsługujemy klikanie przycisków.

Plik: MainWindow.xaml.cs
using System.Collections.ObjectModel;
using System.Windows;
using System.Windows.Controls;

public partial class MainWindow : Window
{
    // Pamiętaj: ObservableCollection to specjalna lista, która informuje ListBox, 
    // gdy dodasz nowy element (lub usuniesz stary). Zwykłe List<T> by tu nie zadziałało!
    public ObservableCollection<ToDoItem> Tasks { get; set; }

    public MainWindow()
    {
        InitializeComponent();
        Tasks = new ObservableCollection<ToDoItem>();

        // BARDZO WAŻNE: Mówimy oknu, że jego źródłem danych bindowania (DataContext) jest ta klasa.
        DataContext = this;
    }

    // Co się dzieje, gdy klikniesz "Dodaj"?
    private void AddBtn_Click(object sender, RoutedEventArgs e)
    {
        // Sprawdzamy, czy ktoś nie wpisał pustego tekstu
        if (!string.IsNullOrWhiteSpace(NewTaskTextBox.Text))
        {
            // Dodajemy element do naszej listy. ListBox w XAML automatycznie to wyłapie!
            Tasks.Add(new ToDoItem { Title = NewTaskTextBox.Text, IsCompleted = false });

            // Wyczyszczenie okienka po dodaniu
            NewTaskTextBox.Clear();
        }
    }

    // Co się dzieje, gdy klikniesz "X" obok zadania?
    private void DeleteBtn_Click(object sender, RoutedEventArgs e)
    {
        // Ponieważ mamy wiele przycisków "X" na liście, zmienna 'sender' pozwala 
        // nam namierzyć konkretnie ten JEDEN przycisk, który kliknięto.
        // Następnie sprawdzamy, jaki obiekt (ToDoItem) jest do niego przypięty (DataContext).
        if (sender is Button btn && btn.DataContext is ToDoItem itemToDelete)
        {
            // Usuwamy zadanie z listy, a ListBox w UI automatycznie znika ten wiersz
            Tasks.Remove(itemToDelete);
        }
    }
}

Dodatek PRO: Interaktywny Canvas i Zdarzenia Myszki

Prowadzący wymaga od Was stworzenia aplikacji interaktywnej. Przejdziemy przez absolutne pozycjonowanie na Canvasie, przeciąganie obiektów myszką (Drag & Drop), dynamiczne menu kontekstowe oraz Data Binding.

10.1. Teoria: Jak działa Canvas i Zdarzenia?

A. Kontrolka Canvas (Płótno)

W przeciwieństwie do <Grid> czy <StackPanel>, <Canvas> nie układa elementów automatycznie. To czyste płótno. Ty decydujesz, w którym pikselu (X, Y) znajdzie się dany kształt (np. Rectangle, Ellipse).

B. Zdarzenia Myszki (Mouse Events)

Prowadzący wymienił zdarzenia: MouseMove, MouseLeftButtonUp, MouseLeftButtonDown. Dlaczego potrzebujemy ich aż trzech?

  • Down (Kliknięcie): Zapisujemy w pamięci, który kształt użytkownik "złapał".
  • Move (Ruch): Jeśli użytkownik trzyma przycisk i rusza myszką, aktualizujemy pozycję (X, Y) złapanego kształtu.
  • Up (Puszczenie): Użytkownik puścił przycisk, więc "upuszczamy" kształt (koniec przeciągania).

C. Interakcja z narysowanymi kształtami

Skąd wiemy, czy kliknęliśmy w puste tło Canvasu, czy w narysowany kwadrat? Używamy właściwości e.OriginalSource. WPF mówi nam dzięki temu: "Zdarzenie kliknięcia dotarło do Canvasu, ale oryginalnie myszka trafiła w ten niebieski kwadrat".

10.2. Projekt: Shape Builder (Gotowiec)

Poniższa aplikacja spełnia wszystkie wymagania z maila: pozwala kliknąć, by narysować kwadrat, złapać go i przesunąć (Drag & Drop), usunąć za pomocą menu kontekstowego, a na górze wyświetla licznik kształtów za pomocą Data Bindingu.

Część 1: XAML - Interfejs i Zdarzenia

Dzielimy okno na górny pasek (wyświetlający bindowany status) oraz główny obszar Canvas. Do Canvasu podpinamy zdarzenia myszki.

Plik: MainWindow.xaml
<Window x:Class="CanvasApp.MainWindow"
        xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
        xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
        Title="Shape Builder" Height="450" Width="600">
    <Grid>
        <Grid.RowDefinitions>
            <RowDefinition Height="Auto"/>
            <RowDefinition Height="*"/>
        </Grid.RowDefinitions>

        <!-- 1. DATA BINDING: Wyświetla zmienną 'StatusText' z klasy C# (np. ile jest kwadratów) -->
        <TextBlock Grid.Row="0" Text="{Binding StatusText}" Padding="10" Background="#333" Foreground="White" FontSize="16"/>

        <!-- 2. CANVAS: Nasłuchujemy 3 zdarzeń myszki.
             Ważne: Tło MUSI być ustawione (nawet na Transparent), inaczej Canvas nie wykryje kliknięcia w puste miejsce! -->
        <Canvas x:Name="DrawCanvas" Grid.Row="1" Background="White"
                MouseLeftButtonDown="Canvas_MouseDown"
                MouseMove="Canvas_MouseMove"
                MouseLeftButtonUp="Canvas_MouseUp">

            <!-- MENU KONTEKSTOWE KANWY (Gdy klikniesz Prawym Przyciskiem Myszki w puste tło) -->
            <Canvas.ContextMenu>
                <ContextMenu>
                    <MenuItem Header="Wyczyść płótno" Click="ClearCanvas_Click"/>
                </ContextMenu>
            </Canvas.ContextMenu>
        </Canvas>
    </Grid>
</Window>

Część 2: Logika w C# (Magia Interakcji)

W tym pliku znajdziesz całą potęgę WPF. Pokazujemy tu, jak tworzyć kształty (Shapes) z poziomu kodu, jak zapamiętać przesunięcie myszki (Offset), aby przeciągany kwadrat nie "skakał" do rogu kursora, oraz jak dynamicznie dopinać do narysowanych kształtów własne Menu Kontekstowe. Zwróć też uwagę na wdrożenie `INotifyPropertyChanged` pod Data Binding.

Plik: MainWindow.xaml.cs
using System.ComponentModel;
using System.Runtime.CompilerServices;
using System.Windows;
using System.Windows.Controls;
using System.Windows.Input;
using System.Windows.Media;
using System.Windows.Shapes;

namespace CanvasApp
{
    public partial class MainWindow : Window, INotifyPropertyChanged
    {
        // == ZMIENNE DO BINDOWANIA ==
        private string statusText = "Kliknij na puste pole, aby narysować kwadrat!";
        public string StatusText
        {
            get => statusText;
            set { statusText = value; OnPropertyChanged(); }
        }

        // == ZMIENNE DO DRAG & DROP (PRZECIĄGANIA) ==
        private bool isDragging = false;     // Czy aktualnie trzymamy przycisk myszy na kształcie?
        private UIElement draggedShape;      // Który dokładnie kształt ciągniemy?
        private Point clickOffset;           // Przesunięcie kursora wewnątrz kształtu (żeby nie skakał do rogu)

        public MainWindow()
        {
            InitializeComponent();
            DataContext = this; // Ustawienie kontekstu dla bindowania StatusText
        }

        // =========================================================
        // 1. MOUSE DOWN (Kliknięcie lewym przyciskiem myszy)
        // =========================================================
        private void Canvas_MouseDown(object sender, MouseButtonEventArgs e)
        {
            // Sprawdzamy w co DOKŁADNIE uderzył kursor (OriginalSource)
            if (e.OriginalSource is Shape clickedShape)
            {
                // SCENARIUSZ A: Kliknęliśmy w narysowany wcześniej kształt
                isDragging = true;
                draggedShape = clickedShape;

                // Zapisujemy, w którym miejscu kształtu złapaliśmy go myszką
                Point mousePos = e.GetPosition(DrawCanvas);
                double shapeLeft = Canvas.GetLeft(draggedShape);
                double shapeTop = Canvas.GetTop(draggedShape);
                clickOffset = new Point(mousePos.X - shapeLeft, mousePos.Y - shapeTop);

                // Złapany obiekt przenosimy na samą górę (żeby nie chował się pod innymi)
                Panel.SetZIndex(draggedShape, 999);

                // Przechwytujemy myszkę (nawet jak wyjedziemy szybko kursorem za kształt, dalej będziemy go trzymać)
                draggedShape.CaptureMouse();

                StatusText = "Przesuwasz kształt...";
            }
            else
            {
                // SCENARIUSZ B: Kliknęliśmy w puste tło Canvasu - Rysujemy nowy kwadrat!
                Point clickPos = e.GetPosition(DrawCanvas);

                Rectangle newRect = new Rectangle
                {
                    Width = 50,
                    Height = 50,
                    Fill = Brushes.DodgerBlue,
                    Stroke = Brushes.DarkBlue,
                    StrokeThickness = 2
                };

                // Ustawiamy pozycję nowego kwadratu w miejscu kliknięcia myszki (centrujemy go)
                Canvas.SetLeft(newRect, clickPos.X - 25);
                Canvas.SetTop(newRect, clickPos.Y - 25);

                // DODAJEMY WŁASNE MENU KONTEKSTOWE DLA TEGO KONKRETNEGO KSZTAŁTU
                ContextMenu shapeMenu = new ContextMenu();
                MenuItem deleteItem = new MenuItem { Header = "Usuń ten kształt" };

                // Używamy wyrażenia lambda by podpiąć funkcję usunięcia wprost do zdarzenia Click
                deleteItem.Click += (s, args) =>
                {
                    DrawCanvas.Children.Remove(newRect);
                    StatusText = $"Liczba kształtów: {DrawCanvas.Children.Count}";
                };

                shapeMenu.Items.Add(deleteItem);
                newRect.ContextMenu = shapeMenu;

                // Wrzucamy kształt na płótno
                DrawCanvas.Children.Add(newRect);
                StatusText = $"Dodano kwadrat! Liczba kształtów: {DrawCanvas.Children.Count}";
            }
        }

        // =========================================================
        // 2. MOUSE MOVE (Ruszanie myszką)
        // =========================================================
        private void Canvas_MouseMove(object sender, MouseEventArgs e)
        {
            // Jeśli trzymamy przycisk i mamy jakiś kształt w garści
            if (isDragging && draggedShape != null)
            {
                Point currentMousePos = e.GetPosition(DrawCanvas);

                // Wyliczamy nową pozycję (obecna myszka MINUS to przesunięcie wewnątrz kształtu)
                double newLeft = currentMousePos.X - clickOffset.X;
                double newTop = currentMousePos.Y - clickOffset.Y;

                // Aktualizujemy położenie kształtu na płótnie
                Canvas.SetLeft(draggedShape, newLeft);
                Canvas.SetTop(draggedShape, newTop);
            }
        }

        // =========================================================
        // 3. MOUSE UP (Puszczenie lewego przycisku)
        // =========================================================
        private void Canvas_MouseUp(object sender, MouseButtonEventArgs e)
        {
            if (isDragging)
            {
                isDragging = false;
                draggedShape.ReleaseMouseCapture(); // Oddajemy myszkę systemowi
                Panel.SetZIndex(draggedShape, 0);   // Zdejmujemy priorytet wyświetlania na wierzchu
                draggedShape = null;

                StatusText = $"Kształt upuszczony. Liczba kształtów: {DrawCanvas.Children.Count}";
            }
        }

        // =========================================================
        // 4. MENU KANWY: Czyszczenie całego płótna
        // =========================================================
        private void ClearCanvas_Click(object sender, RoutedEventArgs e)
        {
            DrawCanvas.Children.Clear();
            StatusText = "Płótno wyczyszczone!";
        }

        // Wymagana implementacja DataBindingu
        public event PropertyChangedEventHandler PropertyChanged;
        protected void OnPropertyChanged([CallerMemberName] string name = null)
        {
            PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(name));
        }
    }
}
