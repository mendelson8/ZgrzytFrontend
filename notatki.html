2. Klasy Danych i INotifyPropertyChanged

Odświeżanie interfejsu (Wskazówka: INotifyPropertyChanged) [cite: 703]
Samo użycie ObservableCollection powiadamia widok tylko o dodaniu lub usunięciu elementu z listy. Aby widok aktualizował się, gdy zmieni się właściwość pojedynczego kontaktu (np. poprawisz nazwisko w edytorze), klasa musi implementować interfejs INotifyPropertyChanged. 

// C#: Contact.cs
using System.ComponentModel;
using System.Runtime.CompilerServices;

public class Contact : INotifyPropertyChanged
{
    private string name;
    private string surname;
    // ... reszta pół ...

    public string Name
    {
        get => name;
        set { name = value; OnPropertyChanged(); }
    }

    public string Surname
    {
        get => surname;
        set { surname = value; OnPropertyChanged(); }
    }

    public Gender Gender { get; set; }

    // Implementacja interfejsu
    public event PropertyChangedEventHandler PropertyChanged;
    protected void OnPropertyChanged([CallerMemberName] string name = null)
    {
        PropertyChanged?.Invoke(this, new PropertyChangedEventArgs(name));
    }
}

4. ListBox (Zmienne Kolory) i DataGrid

Zmienne tło (AlternationIndex) i DataGrid [cite: 593, 628]
Tło dla parzystych i nieparzystych pozycji różni się kolorami. Służy do tego AlternationIndex powiązany z Triggerami w stylach list[cite: 592, 593]. DataGrid musi pozwalać na dodawanie rekordów bezpośrednio z tabeli[cite: 628]. 
<!-- XAML: Wewnątrz ListBox -->
<ListBox ItemsSource="{Binding Source={StaticResource ContactsViewSource}}"
         AlternationCount="2"> <!-- Konieczne do działania zmiennych kolorów! [cite: 625] -->
    <ListBox.ItemContainerStyle>
        <Style TargetType="ListBoxItem">
            <Style.Triggers>
                <Trigger Property="ItemsControl.AlternationIndex" Value="0">
                    <Setter Property="Background" Value="#FFAFC5FF" /> <!-- [cite: 610] -->
                </Trigger>
                <Trigger Property="ItemsControl.AlternationIndex" Value="1">
                    <Setter Property="Background" Value="#FF75A1FF" /> <!-- [cite: 612] -->
                </Trigger>
                <!-- Część domowa: Specjalny kolor dla ZAZNACZONEGO elementu [cite: 641] -->
                <Trigger Property="IsSelected" Value="True">
                    <Setter Property="Background" Value="#FFFDCF6C" />
                </Trigger>
            </Style.Triggers>
        </Style>
    </ListBox.ItemContainerStyle>
</ListBox>

<!-- DataGrid, który sam tworzy kolumny i pozwala na wstawianie wierszy [cite: 630, 631] -->

5. Menu Kontekstowe (Prawy Przycisk)

Usuwanie kontaktu przez prawy przycisk myszy [cite: 643, 644]
Aby po kliknięciu prawym przyciskiem myszy na kontakt pojawiło się menu "Delete contact", należy dodać ContextMenu do struktury DataTemplate naszego ListBoxa[cite: 643, 644]. 
<!-- XAML: Wewnątrz DataTemplate dla ListBoxItem -->
<Grid>
    <Grid.ContextMenu>
        <ContextMenu>
            <!-- Zdarzenie usuwania wiążemy z kodem -->
            <MenuItem Header="Delete contact" Click="MenuItem_DeleteContact" />
        </ContextMenu>
    </Grid.ContextMenu>

    <!-- Tu zawartość kafelka (zdjęcie, imię, nazwisko) -->
</Grid>
// C#: Code-behind (Usuwanie elementu)
private void MenuItem_DeleteContact(object sender, RoutedEventArgs e)
{
    // Pobranie kontekstu klikniętego elementu menu
    if (sender is MenuItem menuItem && menuItem.DataContext is Contact contactToDelete)
    {
        Contacts.Remove(contactToDelete);
    }
}

6. System Walidacji (Interfejsy i Pliki DLL)

Reguły sprawdzające i Interfejs IValidation [cite: 655, 664, 665]
Zadanie wymaga, by można było dodawać reguły walidacji (np. sprawdzanie długości czy formatu XXX-XXX-XXX) bez rekompilacji, używając bibliotek DLL[cite: 663, 664]. Aby to było możliwe, aplikacja i pliki DLL muszą współdzielić interfejs IValidation[cite: 665]. Reguły dziedziczą też po wbudowanej klasie WPF: ValidationRule[cite: 664, 674]. 

// C#: Struktura interfejsu (współdzielona np. w projekcie ClassLibrary) [cite: 665]
public interface IValidation
{
    string Name { get; } // [cite: 666]
    string Description { get; } // [cite: 668]
}

// C#: Przykład Reguły (Sprawdzanie Długości) [cite: 656, 673, 674]
public class MinLengthValidationRule : ValidationRule, IValidation
{
    public string Name => "Minimum 5 znaków";
    public string Description => "Tekst musi posiadać co najmniej 5 znaków.";

    public override ValidationResult Validate(object value, CultureInfo cultureInfo) // [cite: 684]
    {
        string text = value as string;
        if (text == null || text.Length < 5)
        {
            return new ValidationResult(false, Description); // Błąd walidacji
        }
        return ValidationResult.ValidResult; // Walidacja pomyślna [cite: 691]
    }
}

7. Konwertery (Blokada Zakładek)

Zarządzanie widocznością za pomocą mechanizmu konwerterów [cite: 647, 700]
Zakładka "Validation settings" jest zablokowana. Użytkownik odblokowuje ją przyciskiem[cite: 646, 647]. Konwerter w WPF to klasa implementująca interfejs IValueConverter, która potrafi zmienić (np.) stan logiczny bool (True/False) na Visibility (Visible/Collapsed). 
// C#: BoolToVisibilityConverter.cs
using System.Windows.Data;
using System.Windows;

public class BoolToVisibilityConverter : IValueConverter
{
    public object Convert(object value, Type targetType, object parameter, CultureInfo culture)
    {
        if (value is bool isUnlocked && isUnlocked)
            return Visibility.Visible;

        return Visibility.Collapsed;
    }

    public object ConvertBack(object value, Type targetType, object parameter, CultureInfo culture)
    {
        throw new NotImplementedException();
    }
}
<!-- XAML: Wewnątrz Window.Resources -->
<local:BoolToVisibilityConverter x:Key="BoolToVisConverter" />

<!-- XAML: Użycie w kontrolce (np. we wnętrzu TabItem) -->
<!-- Zmienna "IsValidationUnlocked" pochodzi z klasy C# okna głównego -->
<Grid Visibility="{Binding IsValidationUnlocked, Converter={StaticResource BoolToVisConverter}}">
    <!-- Zawartość konfiguracji walidacji -->
</Grid>
