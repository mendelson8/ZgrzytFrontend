Refleksja (Reflection) - Kompendium

Niezbędne usingi:

using System;
using System.Linq;
using System.Reflection;

1. Tworzenie własnych atrybutów

[AttributeUsage(AttributeTargets.Class | AttributeTargets.Method)]
public class DoWyslaniaAttribute : Attribute
{
    public string Opis { get; }
    public DoWyslaniaAttribute(string opis) { Opis = opis; }
}

2. Pobieranie typów oznaczonych atrybutem

var typy = assembly.GetTypes()
    .Where(t => t.GetCustomAttribute<DoWyslaniaAttribute>() != null);

3 - 6. Podstawowe operacje (Skrót)

  • IsSubclassOf: Sprawdzanie dziedziczenia (np. po klasie bazowej).
  • GetProperties: Pobieranie właściwości (można filtrować po atrybutach).
  • Interfejsy Generyczne: Sprawdzanie GetGenericTypeDefinition().
  • SetValue: Ustawianie wartości pól/właściwości w czasie działania (Runtime).

7. Zaawansowane: Metody Generyczne i parametry 'out'

  • MakeGenericMethod: Pozwala wywołać metodę Metoda<T> gdy T znasz dopiero po uruchomieniu programu.
  • MakeByRefType: Niezbędne, aby znaleźć metodę z parametrem ref lub out (np. TryParse).
  • Invoke z tablicą args: Jeśli metoda ma parametr out, wynik zostanie wpisany do tablicy argumentów pod odpowiedni indeks.

8. Praktyczny przykład: Task Runner (Uruchamianie metod z atrybutem)

Ten kod znajduje wszystkie metody w projekcie oznaczone atrybutem [DoWyslania] i je uruchamia. Zwróć uwagę na obsługę metod niestatycznych (konieczność stworzenia instancji).

Kluczowe elementy:

  • SelectMany - Spłaszcza strukturę: zamienia listę klas, gdzie każda ma listę metod, na jedną długą listę wszystkich metod.
  • Activator.CreateInstance - Tworzy obiekt danej klasy, abyśmy mogli wywołać na nim metodę niestatyczną.
  • Invoke(instancja, null) - Pierwszy parametr to obiekt ("this"), na którym wołamy metodę. Dla metod statycznych dajemy null.
public static void RunMarkedMethods()
{
    // KROK A: Pobierz Assembly
    var assembly = Assembly.GetExecutingAssembly();

    // KROK B: Znajdź wszystkie metody z atrybutem [DoWyslania]
    var metodyDoUruchomienia = assembly.GetTypes()   // 1. Wszystkie klasy
        .SelectMany(t => t.GetMethods())             // 2. Wyciągnij wszystkie metody (spłaszczanie)
        .Where(m => m.GetCustomAttributes(typeof(DoWyslaniaAttribute), false).Any())
        .ToList();

    Console.WriteLine($"Znaleziono {metodyDoUruchomienia.Count} metod.\n");

    // KROK C: Uruchom każdą znalezioną metodę
    foreach (var methodInfo in metodyDoUruchomienia)
    {
        var atrybut = (DoWyslaniaAttribute)methodInfo.GetCustomAttribute(typeof(DoWyslaniaAttribute));
        Console.WriteLine($"Opis: {atrybut.Opis}");

        // WAŻNE: Rozróżnienie metod statycznych i instancyjnych
        if (!methodInfo.IsStatic)
        {
            // Musimy stworzyć obiekt klasy, w której jest ta metoda!
            // methodInfo.DeclaringType zwraca typ klasy (np. typeof(Raporty))
            var instancjaKlasy = Activator.CreateInstance(methodInfo.DeclaringType);

            // Invoke(instancja, argumenty)
            methodInfo.Invoke(instancjaKlasy, null);
        }
        else
        {
            // Metoda statyczna nie potrzebuje instancji (null)
            methodInfo.Invoke(null, null);
        }
    }
}

⚠️ Ważne informacje (Egzamin/Rekrutacja)

  • Brak bezpieczeństwa typów: Błędy w nazwach metod (literówki w stringach) powodują błąd dopiero po uruchomieniu (Runtime).
  • Wydajność: Refleksja jest wolna. Nie używaj jej w gorących ścieżkach kodu (hot paths/pętle).
  • Konstruktory: Aby stworzyć instancję przez refleksję, użyj Activator.CreateInstance(typ) lub pobierz konstruktor przez GetConstructor().
  • Prywatne metody: Można je wywołać używając flagi BindingFlags.NonPublic, ale łamie to hermetyzację.
