Transakcje

Transakcja - logiczny ciąg operacji DB zapewniający: atomowość, spójność, izolację, trwałość (ACID).

Składnia (MS SQL):

BEGIN TRANSACTION;
-- operacje DML
COMMIT;   -- zatwierdzenie
ROLLBACK; -- wycofanie

Przykład:

Dodanie rekordu do OrderDetails i aktualizacja pola UnitsInStock dla ProductID = 11 w tabeli Products:

BEGIN TRANSACTION;

-- 1) INSERT do OrderDetails
INSERT INTO OrderDetails (OrderID, ProductID, Quantity)
VALUES (10248, 11, 5);

-- 2) UPDATE w Products
UPDATE Products
SET UnitsInStock = UnitsInStock - 5
WHERE ProductID = 11;

-- 3) Sprawdzenie i COMMIT/ROLLBACK
IF @@ERROR = 0
    COMMIT;
ELSE
    ROLLBACK;

Procedury składowane

Definicja: Procedura składowana - blok kodu SQL przechowywany w bazie, wywoływany z przekazanymi parametrami.

Składnia ogólna

CREATE PROCEDURE dbo.NazwaProcedury
    @Param1 INT,
    @Param2 INT = 0, -- domyślna wartość
    @Param3 NVARCHAR(50) OUTPUT
AS
BEGIN
    SET NOCOUNT ON; -- wyłączanie komunikatu np. (1 row(s) affected)
    -- ciało procedury: INSERT/UPDATE/DELETE/SELECT
END;

Przykłady

1. INSERT

Procedura AddOrderDetail, która przyjmuje OrderID, ProductID i Quantity, dodaje go do tabeli OrderDetails i zwraca w parametrze OUTPUT komunikat sukcesu albo błędu.

CREATE PROCEDURE dbo.AddOrderDetail
    @OrderID INT,
    @ProductID INT,
    @Quantity INT,
    @Message NVARCHAR(100) OUTPUT
AS
BEGIN
    SET NOCOUNT ON;
    BEGIN TRY
        INSERT INTO OrderDetails (OrderID, ProductID, Quantity)
        VALUES (@OrderID, @ProductID, @Quantity);

        SET @Message = N'Wstawiono wiersz pomyślnie';
    END TRY
    BEGIN CATCH
        SET @Message = ERROR_MESSAGE();
    END CATCH
END;

Wywołanie:

DECLARE @Msg NVARCHAR(100);
EXEC dbo.AddOrderDetail
    @OrderID = 10248,
    @ProductID = 11,
    @Quantity = 5,
    @Message = @Msg OUTPUT;

SELECT @Msg AS Message;

2. UPDATE

Procedura przyjmuje OrderID, ProductID, NewQuantity i aktualizuje wartości w tabeli OrderDetails oraz zwraca komunikat.

CREATE PROCEDURE dbo.UpdateOrderDetailQuantity
    @OrderID INT,
    @ProductID INT,
    @NewQuantity INT,
    @Message NVARCHAR(100) OUTPUT
AS
BEGIN
    SET NOCOUNT ON;
    BEGIN TRY
        UPDATE OrderDetails
        SET Quantity = @NewQuantity
        WHERE OrderID = @OrderID AND ProductID = @ProductID;

        -- systemowa zmienna
        IF @@ROWCOUNT = 0
            SET @Message = N'Brak wiersza';
        ELSE
            SET @Message = N'OK';
    END TRY
    BEGIN CATCH
        SET @Message = ERROR_MESSAGE();
    END CATCH
END;

3. DELETE

Procedura przyjmuje OrderID, ProductID i usuwa odpowiadający wiersz z OrderDetails oraz zwraca komunikat.

CREATE PROCEDURE dbo.DeleteOrderDetail
    @OrderID INT,
    @ProductID INT,
    @Message NVARCHAR(100) OUTPUT
AS
BEGIN
    SET NOCOUNT ON;
    BEGIN TRY
        DELETE FROM OrderDetails
        WHERE OrderID = @OrderID AND ProductID = @ProductID;

        IF @@ROWCOUNT = 0
            SET @Message = N'Nie znaleziono wiersza do usunięcia';
        ELSE
            SET @Message = N'Wiersz usunięty pomyślnie';
    END TRY
    BEGIN CATCH
        SET @Message = ERROR_MESSAGE();
    END CATCH
END;

4. UPDATE z REPLACE

Procedura modyfikuje numery telefonów w danym kraju.

CREATE PROCEDURE dbo.UpdateCustomerPhoneByCountry
    @Country NVARCHAR(50),
    @OldChar CHAR(1),
    @NewChar CHAR(1),
    @Message NVARCHAR(100) OUTPUT
AS
BEGIN
    SET NOCOUNT ON;
    BEGIN TRY
        UPDATE Customers
        SET Phone = REPLACE(Phone, @OldChar, @NewChar)
        WHERE Country = @Country;

        IF @@ROWCOUNT = 0
            SET @Message = N'Brak klientów w kraju ' + @Country;
        ELSE
            SET @Message = CAST(@@ROWCOUNT AS NVARCHAR(10)) + N' numerów zmieniono pomyślnie';
    END TRY
    BEGIN CATCH
        SET @Message = ERROR_MESSAGE();
    END CATCH
END;

Tabela logowa i logowanie operacji

Chcemy, żeby każda procedura po wykonaniu dopisywała do osobnej tabeli informacje o wywołaniu.

CREATE TABLE dbo.ChangeLog (
    -- IDENTITY(1, 1) -> zaczynamy od 1, inkrementujemy o 1

Integracja z Javą (JDBC)

Auto-commit

Domyślnie każde DML w JDBC jest od razu zatwierdzane. Aby grupować operacje w transakcję:

try (Connection con = DriverManager.getConnection(url)) {
    con.setAutoCommit(false);
    try {
        // operacje DML lub wywołania procedur
        con.commit();
    } catch (SQLException e) {
        con.rollback();
        throw e;
    }
} catch (Exception e) {
    e.printStackTrace();
}

Przykładowe wywołanie procedury

con.setAutoCommit(false);
String sql = "{ call dbo.AddOrderDetail(?, ?, ?, ?) }";
try (CallableStatement cs = con.prepareCall(sql)) {
    cs.setInt(1, 10248);
    cs.setInt(2, 11);
    cs.setInt(3, 5);
    cs.registerOutParameter(4, Types.NVARCHAR);

    cs.execute();
    System.out.println("Result: " + cs.getString(4));
    con.commit();
} catch (SQLException e) {
    con.rollback();
    e.printStackTrace();
}

Szukanie błędów / niebezpieczeństw w kodzie

Zarządzanie zasobami (Connection/Statement/Scanner)

Błąd: Brak zamknięcia zasobów.

Connection conn = DriverManager.getConnection(connectionUrl);
Statement stmt = conn.createStatement();
Scanner scanner = new Scanner(System.in);
// ... operacje ...
conn.close(); // stmt i scanner wciąż otwarte!

Poprawione (try-with-resources):

try (
    Connection conn = DriverManager.getConnection(connectionUrl);
    Statement stmt = conn.createStatement();
    Scanner scan = new Scanner(System.in)
) {
    // operacje
} catch (SQLException e) {
    e.printStackTrace();
}

Brak transakcji

Jeśli domyślnie autoCommit = true, błąd w jednym zapytaniu nie wycofuje pozostałych. Zawsze używaj con.setAutoCommit(false) oraz con.commit() / con.rollback().

SQL Injection (brak PreparedStatement)

Narażone na SQL Injection (Błąd):

String deleteQuery = "DELETE FROM Products WHERE ProductID = " + deleteId;
stmt.executeUpdate(deleteQuery);
// Jeśli atakujący wprowadzi np. "1; DROP TABLE Products;" tabela zostanie usunięta!

Poprawione (PreparedStatement):

String deleteSql = "DELETE FROM Products WHERE ProductID = ?";
try (PreparedStatement ps = conn.prepareStatement(deleteSql)) {
    ps.setInt(1, deleteId);
    ps.executeUpdate();
}

Zad 2. Procedura Supply

Zlicza ile sztuk zamówiono od ostatniego zapotrzebowania, jeśli > 50, generuje nowe (zaokrąglone do pełnych 10).

CREATE PROCEDURE dbo.Supply
    @ProductID INT
AS
BEGIN
    SET NOCOUNT ON;
    DECLARE @LastRequestDate DATETIME,
            @TotalOrdered INT,
            @RequestQty INT,
            @SupplierID INT;

    -- Pobieranie dostawcy
    SELECT @SupplierID = SupplierID FROM dbo.Products WHERE ProductID = @ProductID;

    -- Ostatnia data zamówienia w ProductRequest
    SELECT @LastRequestDate = MAX(RequestDate) FROM dbo.ProductRequest WHERE ProductID = @ProductID;
    IF @LastRequestDate IS NULL
        SET @LastRequestDate = '1900-01-01';

    -- Suma zamówionych sztuk
    SELECT @TotalOrdered = SUM(od.Quantity)
    FROM dbo.OrderDetails od
    JOIN dbo.Orders o ON o.OrderID = od.OrderID
    WHERE od.ProductID = @ProductID AND o.OrderDate > @LastRequestDate;

    IF @TotalOrdered IS NULL SET @TotalOrdered = 0;

    -- Jeśli > 50, dodaj nowe
    IF @TotalOrdered > 50
    BEGIN
        SET @RequestQty = CEILING(@TotalOrdered / 10.0) * 10;
        INSERT INTO dbo.ProductRequest (ProductID, SupplierID, Quantity, RequestDate)
        VALUES (@ProductID, @SupplierID, @RequestQty, GETDATE());
    END
END;

Zad 3. Wyzwalacz OrderTrigger

CREATE TRIGGER dbo.OrderTrigger
ON dbo.OrderDetails
AFTER INSERT
AS
BEGIN
    SET NOCOUNT ON;
    DECLARE @pid INT;

    -- Kursor dla unikalnych ProductID z wstawionych wierszy
    DECLARE prod_cur CURSOR LOCAL FAST_FORWARD FOR
    SELECT DISTINCT ProductID FROM inserted;

    OPEN prod_cur;
    FETCH NEXT FROM prod_cur INTO @pid;

    WHILE @@FETCH_STATUS = 0
    BEGIN
        EXEC dbo.Supply @ProductID = @pid;
        FETCH NEXT FROM prod_cur INTO @pid;
    END

    CLOSE prod_cur;
    DEALLOCATE prod_cur;
END;

    // Metoda pomocnicza do zliczania rekordów
    private static void printCount(Statement stmt, String table, String label) throws SQLException {

        }
    }
